# Channel v0.4 key 対応

## 背景

`.workspace/plan-20260530.txt` の v0.4 設計準拠対応では、DDB item schema の差分解消が残っている。`docs/design/dynamodb-schema-audit.md` では `Channel` が `CH#{channel_id}` / `META` として設計されている一方、現行 `put_channel` は `CHANNEL#{channel_id}` / `META` へ保存している。

## 目的

対象チャンネル設定の新規保存 key を v0.4 の `CH#{channel_id}` / `META` に寄せ、既存 `CHANNEL#{channel_id}` / `META` の読み取り互換を維持する。

## タスク種別

機能追加

## スコープ

- `apps/shared/src/diopside_core/repository.py` の `Channel` writer/get。
- `apps/api/src/diopside_api/handler.py` の既存管理 channel API contract に影響しないことの確認。
- `tests/test_repository_schema_contract.py`、`tests/test_api_handler.py`、`tests/test_core_pipeline.py` の contract 更新。
- `README.md` と `docs/design/dynamodb-schema-audit.md` の `Channel` 形状記述。
- 作業レポート、PR コメント、task done 更新。

## スコープ外

- 既存 DynamoDB data の backfill。
- ChannelRef rebuild job。
- 管理 UI からの新規 AppConfig/Channel 一括編集。

## 受け入れ条件

- [x] `put_channel` が `pk=CH#{channel_id}` / `sk=META` の `Channel` item を保存する。
- [x] `Channel` item が `channel_id`、`channel_title`、`uploads_playlist_id`、`collect_enabled`、`default_tags` を持つ。
- [x] `get_channel` が新 key を優先し、旧 `CHANNEL#...` / `META` も fallback で扱える。
- [x] `ChannelRef` は引き続き `APP#CHANNELS` / `CH#{channel_id}` に同期され、管理 channel API が通る。
- [x] `README.md` と `docs/design/dynamodb-schema-audit.md` が実装済み形状に同期している。
- [x] 選定した検証コマンドが pass し、未実施の検証がある場合は理由を記録する。
