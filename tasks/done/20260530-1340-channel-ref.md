# ChannelRef read model

## 背景

`.workspace/plan-20260530.txt` は v0.4 DDB item schema への準拠を検収基準に戻す方針を示している。
v0.4 の `ChannelRef` は `pk=APP#CHANNELS`, `sk=CH#{channel_id}` で、対象チャンネル一覧を Scan なしに列挙する read model である。
現状は `put_channel` が `Channel` item のみを保存し、`DynamoRepository.list_channels()` は table scan 相当で `Channel` item を列挙している。

## 目的

チャンネル設定更新時に `ChannelRef` を保存し、`GET /api/admin/channels` が read model 経由でチャンネル一覧を返せる状態にする。

## タスク種別

機能追加

## スコープ

- repository に `ChannelRef` item type、writer、query path を追加する。
- `put_channel` 時に `Channel` と `ChannelRef` を同期する。
- `list_channels` が保存済み `ChannelRef` を優先利用し、空の場合は既存 `Channel` 一覧 fallback を維持する。
- DynamoDB adapter の `list_channels` を Scan から `pk=APP#CHANNELS` Query 優先へ変更する。
- 管理チャンネル一覧 API の contract test、README、traceability、DDB audit、compliance audit を更新する。

## 対象外

- `Channel` item 本体の v0.4 key prefix `CH#{channel_id}` への全面移行。
- `ChannelSyncCursor` key prefix 移行。
- 既存 DynamoDB data の `ChannelRef` backfill job。
- 管理 UI のチャンネル編集画面拡張。
