# VideoTagLink item 対応

## 背景

`.workspace/plan-20260530.txt` の v0.4 設計準拠対応では、DDB item schema の差分解消が残っている。`docs/design/dynamodb-schema-audit.md` では `VideoTagLink` が `VID#{video_id}` / `TAG#{tag_id}` として設計されている一方、現行実装は `VideoTagIndex` の `TAG#{tag}` / `VIDEO#{video_id}` だけを保存している。

## 目的

`put_video` とタグ補正経路で v0.4 の `VideoTagLink` item を保存・削除し、既存 `VideoTagIndex` 互換を維持したまま tag link 正本へ近づける。

## タスク種別

機能追加

## スコープ

- `apps/shared/src/diopside_core/repository.py` の `VideoTagLink` allowlist / writer / delete。
- `tests/test_repository_schema_contract.py` の DDB schema contract。
- `README.md` と `docs/design/dynamodb-schema-audit.md` の VideoTagLink 形状記述。
- 作業レポート、PR コメント、task done 更新。

## スコープ外

- 既存 DynamoDB data の backfill。
- tag_id 正規化辞書や tag category 管理 UI。
- tag search/list の `VideoTagLink` GSI への全面切替。

## 実施計画

1. v0.4 の `VideoTagLink` item schema と現 `VideoTagIndex` writer を確認する。
2. `video_tag_link_item` helper を追加し、`ITEM_TYPES` に `VideoTagLink` を追加する。
3. `put_video` が現 `VideoTagIndex` に加えて `VideoTagLink` を保存し、削除タグでは両方を消すようにする。
4. schema contract test と audit / README を更新する。
5. targeted test、docs consistency、diff check、全体 verify を実行する。

## ドキュメントメンテナンス方針

`README.md` の item schema 表と `docs/design/dynamodb-schema-audit.md` の `VideoTagLink` 行を更新する。設計書本体は既に v0.4 形状を記載しているため、audit 側を実装済みに寄せる。
