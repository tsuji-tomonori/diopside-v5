# Video v0.4 key 対応

## 背景

`.workspace/plan-20260530.txt` の v0.4 設計準拠対応では、DDB item schema の差分解消が残っている。`docs/design/dynamodb-schema-audit.md` では `Video` が `VID#{video_id}` / `META` として設計されている一方、現行 `video_item` は `VIDEO#{video_id}` / `META` へ保存している。

## 目的

公開動画 metadata の新規保存 key を v0.4 の `VID#{video_id}` / `META` に寄せ、既存 `VIDEO#{video_id}` / `META` の読み取り fallback を維持する。

## タスク種別

機能追加

## スコープ

- `apps/shared/src/diopside_core/repository.py` の `Video` writer/get/list。
- `tests/test_repository_schema_contract.py` と関連 API / worker / static exporter tests。
- `README.md` と `docs/design/dynamodb-schema-audit.md` の `Video` 形状記述。
- 作業レポート、PR コメント、task done 更新。

## スコープ外

- 既存 DynamoDB data の backfill。
- legacy `VideoTagIndex` の key 移行。
- `VideoStateEvent` / `VideoStatSnapshot` の追加。
- public export JSON schema の大規模変更。

## 受け入れ条件

- [x] `video_item` / `put_video` が `pk=VID#{video_id}` / `sk=META` の `Video` item を保存する。
- [x] public `Video` item が `gsi1pk=VIDEO#PUBLIC` と v0.4 prefix の `gsi1sk=PUB#...` を持つ。
- [x] `get_video` が新 key を優先し、旧 `VIDEO#...` / `META` も fallback で扱える。
- [x] `VideoMonthIndex`、`VideoTagLink`、`RandomBucket`、legacy `VideoTagIndex` の既存 contract が維持される。
- [x] `README.md` と `docs/design/dynamodb-schema-audit.md` が実装済み形状に同期している。
- [x] 選定した検証コマンドが pass し、未実施の検証がある場合は理由を記録する。
- [x] PR に受け入れ条件確認コメントとセルフレビューコメントを日本語で追加する。
