# XSS対策維持

状態: do

## 背景

`.workspace/plan-20260529.txt` の P3-06 に従い、public JSON 由来の文字列は `textContent` または属性設定で扱い、`innerHTML` を使わない方針を維持する。

## 目的

公開 UI の DOM 生成方針を静的検証できるようにし、危険な HTML sink が混入した場合に `npm test` で検出する。

## タスク種別

セキュリティ検証追加

## スコープ

- `tools/`
- `package.json`
- 必要に応じて `apps/web/public/app.js`

## 計画

1. 既存 public UI の DOM 更新箇所を確認する。
2. `innerHTML`、`outerHTML`、`insertAdjacentHTML`、`document.write` などの禁止 sink を検出する checker を追加する。
3. checker を `npm test` に組み込む。
4. 検証、作業レポート、commit、PR、受け入れ条件コメント、セルフレビューを完了する。

## ドキュメント保守方針

セキュリティ方針は plan に明記されており、今回の変更は自動検証追加。README 更新は不要で、作業レポートに検証内容を残す。

## 受け入れ条件

- public UI が `innerHTML` を使っていないことを検証できる。
- public UI が危険な HTML sink を使っていないことを検証できる。
- checker が `npm test` で実行される。
- 既存の DOM 生成 helper が `textContent` / 属性設定方針を維持する。
- 変更範囲に見合う検証と `npm test` が成功する。

## 検証計画

- `git diff --check`
- `node tools/check-web-dom-safety.mjs`
- `npm test`
- `npm run build`
- `npm run e2e:local`

## リスク

- 静的な文字列検出であり、ブラウザ runtime sanitizer の証明ではない。禁止 sink 混入の regression guard として扱う。
