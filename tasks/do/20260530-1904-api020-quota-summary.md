# API-020 quota daily summary response

状態: in_progress

タスク種別: 機能追加

## 背景

`.workspace/plan-20260530.txt` の v0.4 設計準拠対応では、API-020 `GET /api/admin/quota-usage` が日別使用量、メソッド別使用量、日次上限、警告メッセージを返すことを求めている。現実装は call record の `items` のみを返しており、BATCH-016 の daily summary / warning metadata を API から確認できない。

## 目的

管理 API の quota usage response に v0.4 の `daily`、`by_method`、`limit_per_day`、`warning` を追加し、既存 UI/API 互換の `items` は維持する。

## スコープ

- repository の daily quota summary list helper。
- `GET /api/admin/quota-usage` の response shape 拡張。
- `from` / `to` / `method` query による summary / call record filter。
- API handler tests、README / audit docs、作業完了レポート。

## スコープ外

- 管理 UI の quota panel 表示拡張。
- BATCH-016 以外での daily summary backfill。
- CloudWatch Alarm / 外部通知 delivery。

## 計画

1. repository に `list_quota_summaries` を追加する。
2. handler の quota response を `items` 互換 + v0.4 fields に拡張する。
3. tests と docs を更新する。
4. targeted checks と `npm run verify` を実行する。
5. PR コメント、task done、作業レポートまで完了する。

## PR レビュー観点

- 既存の `items` 互換を壊していないこと。
- v0.4 の `daily` / `by_method` が daily summary 由来であり、call record と混同していないこと。
- warning 表示が summary metadata に基づき、未実施の外部通知を実施済み扱いしていないこと。

## リスク

- daily summary が存在しない既存環境では `daily` / `by_method` は空になり、`items` の call record だけが返る。backfill は後続対象。
