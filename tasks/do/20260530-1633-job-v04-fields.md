# Job v0.4 field alignment

状態: do

## 背景

`.workspace/plan-20260530.txt` と `.workspace/` の設計書対応の一環として、DynamoDB schema audit で `Job` item が部分実装として残っている。現状は `dedupe_key` ではなく `idempotency_key`、`latest_state` ではなく `derived_state` を中心に保存している。

## 目的

`Job` item を basic design v0.4 のフィールド表現へ寄せ、既存の job 作成・イベント追加・一覧取得の挙動を維持する。

## タスク種別

機能追加

## スコープ

- `Job` item の `dedupe_key` / `latest_state` / state index 表現を追加・整合する。
- job event 追加時に親 `Job` item の最新状態を更新する。
- memory repository と Dynamo repository の job get/list 挙動を維持する。
- schema contract テスト、README、DynamoDB schema audit を更新する。

## 計画

1. repository の job 作成・イベント追加・一覧取得実装を確認する。
2. `Job` item に v0.4 alias field と state index を追加する。
3. job event 追加時に `latest_state` / `derived_state` / GSI key を更新する。
4. list/get の既存利用者が壊れないようにテストを更新する。
5. README と schema audit の残課題表を更新する。
6. 対象検証と `npm run verify` を実行する。

## ドキュメント保守方針

README の実装済み/未実装リストと `docs/design/dynamodb-schema-audit.md` の `Job` 行を実装状況に合わせて更新する。API 仕様や運用手順の変更は想定しない。

## 受け入れ条件

- `create_job` が `Job` item に `dedupe_key` と `idempotency_key` を保存する。
- `create_job` が `latest_state` と `derived_state` の初期値 `queued` を保存する。
- `Job` item の GSI3 key が `JOB#STATE#{latest_state}` / `NEXT#{next_run_at}#{job_id}` 形式になる。
- `append_job_event` 後に親 `Job` item の `latest_state` と `derived_state` が派生状態へ更新される。
- Memory repository と Dynamo repository の `get_job` / `list_jobs` が既存用途で利用できる。
- README と DynamoDB schema audit が更新済みである。
- 選定した検証コマンドが pass し、未実施検証があれば理由を記録する。

## 検証計画

- `python3 -m py_compile apps/shared/src/diopside_core/repository.py`
- `PYTHONPATH=apps/shared/src python3 -m pytest tests/test_repository_schema_contract.py`
- 必要に応じて job/list 周辺の関連 pytest
- `node tools/check-docs-consistency.mjs`
- `git diff --check`
- `npm run verify`

## PR レビュー観点

- `Job` item の v0.4 field alias が既存互換を壊していないこと。
- `list_jobs` が state index 変更後も必要な state を返すこと。
- docs と tests が実装差分に追従していること。

## リスク

- DynamoDB の既存 item が旧 `JOB#ALL` GSI key を持つ場合があるため、互換 fallback を残す必要がある。
- job state の派生値は event log に依存するため、親 item 更新と `get_job` 派生表示の整合を確認する必要がある。
