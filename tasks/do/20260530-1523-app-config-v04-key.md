# AppConfig v0.4 key 対応

## 背景

`.workspace/plan-20260530.txt` の v0.4 設計準拠対応では、DDB item schema の差分解消が残っている。`docs/design/dynamodb-schema-audit.md` では `AppConfig` が `APP#CONFIG` / `META` として設計されている一方、現 README と schema contract は旧 `CONFIG#app` key を扱っている。

## 目的

アプリ全体設定の新規保存 key を v0.4 の `APP#CONFIG` / `META` に寄せ、既存 `CONFIG#app` / `META` の読み取り fallback を維持する。

## タスク種別

機能追加

## スコープ

- `apps/shared/src/diopside_core/repository.py` の `AppConfig` writer/get。
- `tests/test_repository_schema_contract.py` の schema contract。
- `README.md` と `docs/design/dynamodb-schema-audit.md` の `AppConfig` 形状記述。
- 作業レポート、PR コメント、task done 更新。

## スコープ外

- 既存 DynamoDB data の backfill。
- `GET /api/config` を DDB 設定読み込みへ切り替えること。
- secret 値や SSM Parameter 値の保存。
- 管理 UI からの AppConfig 編集。

## 実施計画

1. 現行 `AppConfig` contract と v0.4 schema を確認する。
2. `app_config_item` helper と `put_app_config` / `get_app_config` を repository に追加する。
3. `get_app_config` は新 key を優先し、旧 `CONFIG#app` / `META` を fallback する。
4. schema contract test と audit / README を更新する。
5. targeted test、docs consistency、diff check、全体 verify を実行する。

## ドキュメントメンテナンス方針

`README.md` の item schema 表と `docs/design/dynamodb-schema-audit.md` の `AppConfig` 行を更新する。設計書本体は既に v0.4 形状を記載しているため、audit 側を実装済みに寄せる。

## 受け入れ条件

- [ ] `put_app_config` が `pk=APP#CONFIG` / `sk=META` の `AppConfig` item を保存する。
- [ ] `AppConfig` item が `system_name`、`target_channel_ids`、`youtube_api_key_ssm_param`、`collection_enabled`、`public_export_enabled`、`default_locale`、`public_base_path` を持つ。
- [ ] `get_app_config` が新 key を優先し、旧 `CONFIG#app` / `META` も fallback で扱える。
- [ ] secret 値を保存せず、SSM Parameter 名のみを field として扱う。
- [ ] `README.md` と `docs/design/dynamodb-schema-audit.md` が実装済み形状に同期している。
- [ ] 選定した検証コマンドが pass し、未実施の検証がある場合は理由を記録する。
- [ ] PR に受け入れ条件確認コメントとセルフレビューコメントを日本語で追加する。

## 検証計画

- `python3 -m py_compile apps/shared/src/diopside_core/repository.py`
- `PYTHONPATH=apps/shared/src python3 -m pytest tests/test_repository_schema_contract.py`
- `node tools/check-docs-consistency.mjs`
- `git diff --check`
- `npm run verify`

## PR レビュー観点

- v0.4 key へ新規保存しつつ、旧 config の読み取り互換を壊していないこと。
- YouTube API key などの secret 値を DynamoDB item に追加していないこと。
- API runtime の設定読み込み切替を実施済みと誤記していないこと。

## リスク

- 既存 DynamoDB data への backfill は未実施。
- `GET /api/config` は現時点では static public config response のままで、DDB AppConfig 読み込みには未接続。
