- 既存の `append_job_event(job_id, event_type, details)` 呼び出しを変えると worker/API 側の影響が広がるため、引数は維持して writer 内で v0.4 item に変換した。
- 管理 UI/API の既存表示が `event_type` / `details` を参照しているため、正本 field は `event_name` / `state_after` / `payload` としつつ、互換 alias として旧 field も保持した。
- 既存 DynamoDB に `EVENT#...` item が残る可能性を考慮し、`get_job` では旧 shape を正規化して状態導出できるようにした。
- 高並列 append 時の厳密な `seq` 採番は DynamoDB 条件付き書き込みや専用 counter 設計が必要なため、今回の範囲では既存 event からの採番に留め、リスクとして記録した。
