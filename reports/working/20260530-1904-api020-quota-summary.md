`.workspace/plan-20260530.txt` と `.workspace/` 配下の設計書に基づき、`main` を pull した上で v0.4 設計準拠対応を進める。

## 要件整理

- API-020 は quota 使用量 API として `daily`、`by_method`、`limit_per_day`、`warning` を返す必要がある。
- 既存管理 UI は call record `items` を読んでいるため互換を維持する。
- BATCH-016 の daily summary が未作成の既存環境でも API が壊れないようにする。

## 検討・判断

repository に daily summary list helper を追加し、API handler 側で v0.4 response shape に整形した。`items` は既存互換の call record として残し、`daily` / `by_method` は `QuotaUsage` daily summary 由来に限定した。

`from`、`to`、`method` query は call record と daily summary の両方に適用する。daily summary がない環境では `daily` / `by_method` は空配列、`warning` は `null` とし、既存 UI の call record 表示を維持する。

## 実施作業

- `MemoryRepository` / `DynamoRepository` に `list_quota_summaries` を追加した。
- `GET /api/admin/quota-usage` を `items` 互換 + `daily` / `by_method` / `limit_per_day` / `warning` response へ拡張した。
- query filter と warning message の tests を追加した。
- README、traceability、compliance audit を更新した。
