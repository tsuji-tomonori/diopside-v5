# Video v0.4 key 対応 作業レポート

## 受けた指示

- `.workspace/plan-20260530.txt` と `.workspace/` の設計書に沿って v0.4 設計準拠を進める。
- main を pull してから worktree / task / PR flow で作業する。

## 要件整理

- `Video` の新規保存 key を `VID#{video_id}` / `META` にする。
- public GSI sort key を v0.4 の `PUB#{inverted_published_at}#{video_id}` 形状に寄せる。
- 既存 `VIDEO#{video_id}` / `META` は読み取り fallback として維持する。
- `VideoMonthIndex`、`VideoTagLink`、`RandomBucket`、legacy `VideoTagIndex` の既存 contract は維持する。
- README と schema audit を実装済み範囲に同期する。

## 検討・判断

- `video_item` の primary key を `VID#{video_id}` / `META` に変更した。
- `get_video` は新 key を優先し、旧 `VIDEO#{video_id}` / `META` を fallback する。
- public GSI は `gsi1sk=PUB#{inverted_published_at}#{video_id}` に変更し、DynamoDB query は昇順で最新順になるよう調整した。
- legacy `VideoTagIndex` は互換維持のため `TAG#{tag}` / `VIDEO#{video_id}` を継続した。

## 実施作業

- `inverted_timestamp` helper を追加し、`video_item` の key / public GSI shape を更新した。
- `put_video` が既存 item を新旧 key fallback で merge してから保存するよう更新した。
- repository schema contract、API/core/static exporter 関連 tests の対象範囲を確認した。
- `README.md` と `docs/design/dynamodb-schema-audit.md` の `Video` 記述を更新した。
