# ChannelRef read model 作業レポート

## 指示

- `.workspace/plan-20260530.txt` の v0.4 設計準拠対応を継続する。
- main を pull してから、Worktree Task PR Flow に従い task md、実装、検証、レポート、PR 反映まで進める。

## 要件整理

- v0.4 の `ChannelRef` item を `pk=APP#CHANNELS`, `sk=CH#{channel_id}` で保存する。
- `put_channel` で `Channel` と `ChannelRef` を同期する。
- 管理チャンネル一覧は `ChannelRef` read model を優先し、未 backfill 環境では既存 `Channel` fallback を維持する。
- `DynamoRepository.list_channels` は Scan ではなく `pk=APP#CHANNELS` Query path を優先する。
- 既存 DynamoDB data の backfill、`Channel` 本体の v0.4 key prefix 移行、管理 UI 拡張は対象外。

## 実施作業

- root `main` で `git pull --ff-only` を実行し、`Already up to date` を確認した。
- repository に `ChannelRef` item type と `channel_ref_item` を追加した。
- `put_channel` が `Channel` 保存後に `ChannelRef` を同期保存するようにした。
- `MemoryRepository.list_channels` は `ChannelRef` を優先し、空の場合のみ `Channel` fallback を使うようにした。
- `DynamoRepository.list_channels` は `pk=APP#CHANNELS` / `sk begins_with CH#` の Query を優先し、空の場合のみ fallback を使うようにした。
- 管理 API の channel response は `ChannelRef` の `collect_enabled` / `channel_title` からも既存 `enabled` / `display_name` を返せるようにした。
- repository/API tests、README、traceability matrix、DDB schema audit、compliance audit を更新した。
