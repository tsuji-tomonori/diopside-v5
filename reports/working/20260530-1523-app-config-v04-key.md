# AppConfig v0.4 key 対応 作業レポート

## 受けた指示

- `.workspace/plan-20260530.txt` と `.workspace/` の設計書に沿って v0.4 設計準拠を進める。
- main を pull してから worktree / task / PR flow で作業する。

## 要件整理

- `AppConfig` の新規保存 key を `APP#CONFIG` / `META` にする。
- v0.4 required fields を repository writer で補完する。
- 既存 `CONFIG#app` / `META` は読み取り fallback として維持する。
- secret 値は保存せず、SSM Parameter 名のみを保存対象にする。
- README と schema audit を実装済み範囲に同期する。

## 検討・判断

- `put_app_config` / `get_app_config` を repository に追加し、アプリ全体設定の v0.4 key 保存を固定した。
- `youtube_api_key` のような secret 値は `app_config_item` で除外し、`youtube_api_key_ssm_param` のみを保存する。
- `GET /api/config` の runtime 接続は public config response の挙動変更になるため、今回の scope から外した。

## 実施作業

- `app_config_item`、`put_app_config`、`get_app_config` を追加した。
- schema contract test に v0.4 key、required fields、secret 除外、旧 key fallback を追加した。
- `README.md` と `docs/design/dynamodb-schema-audit.md` の `AppConfig` 記述を更新した。
